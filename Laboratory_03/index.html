Javadoc

Javadoc is a tool dedicated to generating documents in an HTML format. This format is similar to the Java API and is created in the same way. When Javadoc compiles, it looks for specific comments to fill in the html document. The way to formally write to javadoc in a document is using the: 


/**
 * Method/Class described here    
 */
comment before the method that it will be describing.

Javadoc

Describing the parts of a method is through the use of the keywords, 


/**
 *@param VARIABLE_NAME Description of variable
 *@author CREATORSNAME
 *@return Description of returned value
 */

Listeners

Every Graphical User Interface that incorporates a response to user actions must incorporate a type of listener. Different types include Mouse, Timer, and Key listeners. Just as it sounds, listeners “listen” or wait for a specific action described to do a defined task.

Listeners

  1. They are user created classes which implement the Interface ActionListener (which must be imported from the java.awt.event library)
  2. The interface forces the class to contain a method called actionPerformed(ActionEvent event). The parameter ActionEvent must also be imported from java.awt.event library.
  3. If you forget to add listeners to an object like Buttons or Frames, an error will not be generated but such items will do nothing in the application.
